GitHub Dual Account SSH + Signing

Minimal command reference for running two GitHub accounts with verified commits.

1. Primary Account (Global Setup)

git config --global user.name "YourPrimaryUser"
git config --global user.email "01234567+YourPrimaryUser@users.noreply.github.com"
git config --global gpg.format ssh
git config --global commit.gpgsign true
git config --global tag.gpgsign true
git config --global user.signingkey ~/.ssh/id_ed25519.pub

2. Secondary Account (Per Repository)

git config user.name "YourSecondUser"
git config user.email "01234567+YourSecondUser@users.noreply.github.com"
git config user.signingkey ~/.ssh/id_ed25519_second.pub
git config gpg.format ssh
git config commit.gpgsign true

3. SSH Remote

git remote add origin git@github.com:YourSecondUser/your-repository.git

4. GitHub CLI Authentication

gh auth login

CLI authentication is separate from SSH authentication and commit signing. Choose SSH protocol when prompted.

5. Repository Overrides Global Config

.git/config

Each repository can override:

user.name
user.email
user.signingkey

6. Verification / Debug Commands

git config --list --show-origin
git config --show-origin user.signingkey
git commit --allow-empty -m "signing test"
git commit -S --allow-empty -m "forced signing test"
git log -1 --show-signature
git log -1 --pretty=full
ssh -T git@github.com
git remote -v
